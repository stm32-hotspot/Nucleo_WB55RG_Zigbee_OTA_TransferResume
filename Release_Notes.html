Purpose

This STM32WB Zigbee OTA example is based on STM32WB Zigbee OTA Client Router Example and can be tested along with the STM32WB Zigbee OTA Server Coordinator Example to push an OTA update image to the client.

This example bring additional features to the legacy project : Pause/resume OTA transfet, recover OTA transfert after power off/ on cycle , OTA downgrade prevention.

Update History

Main Changes

Initial release V1.0 of STM32WB Zigbee OTA Client Router Improved Example

Introduced new features over legacy example:

  • Pause/resume the OTA image download by using SW1 (to pause) or SW3 (to resume) buttons.
  • OTA transfer resume after a reset/power off (ideal for SED or energy harvester devices willing to resume download when V battery is good again)
  • OTA downgrade prevention

Tested on SDK V1.16.

STM32WB Zigbee OTA example is compiled with STM32CubeIDE toolchain gnu 11 and gnu++ 14 tool to initialize the system, peripherals and middleware stacks. STM32CubeIDE.

Known Limitations

  • SRP cache on the OpenThread border router needs to be flushed each time a new commissioning process is attempted. For this, factory reset the device by pressing SW1 or B1 for at least 10sec, then do a hardware reset then reset the RCP device or execute in the raspberry terminal ot-ctl srp server disable ; ot-ctl srp server enable, then restart the Android application . After this step, the commissioning process of the device can start;
  • Due to some MDNS issues, the commissioning with some Wi-Fi access point may fail.
  • Numerous compilation warnings, mostly in the Matter SDK.
  • The upstream application build is supported on Windows, MacOS and Linux. However the package itself has only been tested on a Windows 10 host.
